# Требования к функционалу «Мои места» в личном кабинете

## Введение и контекст

### Краткое описание проекта

**Название проекта:** FishMap — платформа для рыболовов

**Тип проекта:** Микросервисное веб-приложение для управления местами для рыбалки

**Технический стек:**
- **Frontend:** Next.js 15 (App Router), React 18, TypeScript, Tailwind CSS, Zustand
- **Backend:** FastAPI (Python), микросервисная архитектура
- **База данных:** PostgreSQL 14+ с общей базой для всех сервисов
- **Инфраструктура:** Docker Swarm, Traefik (API Gateway)
- **Карта:** Яндекс Карты JavaScript API (API Key: dfb59053-0011-47fb-a6f1-a14efb9160d1)

**Архитектурные компоненты:**
- Auth Service (порт 8001) — аутентификация и пользователи
- Places Service (порт 8002) — управление местами для рыбалки
- Reports Service (порт 8003) — отчеты рыбаков
- Booking Service (порт 8004) — бронирование
- Shop Service (порт 8005) — магазин
- Email Service (порт 8006) — почтовые уведомления

**Текущее состояние:**
- Таблица `places` в базе данных создана
- Frontend компоненты `MyPlacesTab`, `MapPage`, `PlacePicker`, `YandexMap` созданы
- Places Service имеет базовый CRUD API
- Интеграция с Яндекс Картами реализована на базовом уровне

### Сценарии использования (Use Cases)

#### UC1: Добавление личного места кликом на карту
**Актор:** Авторизованный пользователь

**Основной поток:**
1. Пользователь авторизуется в системе
2. Открывает вкладку «Мои места»
3. Нажимает кнопку «Добавить место»
4. Кликает на карту в нужном месте
5. Появляется баллон с вопросом «Добавить место?»
6. Нажимает кнопку «Добавить»
7. Открывается модальная форма с автоматически заполненными координатами
8. Система автоматически заполняет адрес, город, регион (через Яндекс Карты)
9. Пользователь заполняет остальные поля
10. Выбирает тип места (База отдыха, Дикое место, Кемпинг)
11. Если выбран «База отдыха» или «Кемпинг» — отображаются дополнительные поля
12. Нажимает «Сохранить»
13. Система сохраняет место и показывает подтверждение

**Альтернативные потоки:**
- 5a. Пользователь нажимает «Отмена» в баллоне — форма не открывается
- 9a. Пользователь вводит данные вручную в поля координат
- 10a. Если выбрано «Публичное» — место отправляется на модерацию

---

#### UC2: Просмотр списка личных мест
**Актор:** Авторизованный пользователь

**Основной поток:**
1. Пользователь авторизуется
2. Открывает вкладку «Мои места»
3. Видит список всех сохраненных мест
4. Для каждого места отображается:
   - Фото (первое изображение)
   - Название
   - Тип места (иконка)
   - Адрес
   - Типы рыб (теги)
   - Рейтинг (для публичных)
   - Статус модерации (для публичных)
   - Дата последнего посещения
5. Пользователь может:
   - Открыть место на карте
   - Редактировать место
   - Удалить место
   - Поделиться местом
   - Добавить в избранное

**Альтернативные потоки:**
- 3a. Если мест нет — отображается пустое состояние
- 5a. Если место публичное и на модерации — действие «Поделиться» недоступно

---

#### UC3: Просмотр личных мест на карте
**Актор:** Авторизованный пользователь

**Основной поток:**
1. Пользователь открывает вкладку «Мои места»
2. Включает режим «Показать карту»
3. На карте отображаются все личные места пользователя
4. При наведении на точку отображается баллон с информацией:
   - Название
   - Первое фото
   - Типы рыб (теги)
   - Рейтинг (если место публичное)
   - Комментарий (первый из описания)
5. При клике на точку — открывается детальная страница места

## Функциональные требования

### Модуль 1: Добавление места на карте (UX улучшение)

#### FR1.1: Быстрое добавление места кликом на карту

**User Story:**
> Как авторизованный пользователь, я хочу добавить место кликом на карту с последующим подтверждением, чтобы сэкономить время на ручном вводе координат и получить удобный UX.

**Критерии приемки (Acceptance Criteria):**

**Дано (Given):**
- Пользователь авторизован
- Открыта вкладка «Мои места»
- Карта отображена на странице

**Когда (When):**
- Пользователь нажимает кнопку «Добавить место»
- Включается режим добавления (курсор меняется на перекрестие)
- Пользователь кликает на карту в нужном месте

**Тогда (Then):**
- На карте появляется временный маркер в точке клика
- Отображается баллон с текстом: «Добавить место?»
- В баллоне две кнопки: «Добавить» и «Отмена»
- При нажатии «Добавить»:
  - Открывается модальная форма создания места
  - Координаты автоматически заполняются из точки клика
  - Выполняется обратное геокодирование через Яндекс Карты
  - Адрес, город, регион автоматически заполняются (если данные доступны)
- При нажатии «Отмена»:
  - Маркер исчезает
  - Форма не открывается

**Валидация:**
- Координаты должны быть валидны (широта: -90 до 90, долгота: -180 до 180)
- Обратное геокодирование выполняется асинхронно
- Если адрес не определен — поля адреса/города/региона остаются пустыми

#### FR1.2: Типизация мест (3 типа)

**User Story:**
> Как пользователь, я хочу выбрать тип места (База отдыха, Дикое место, Кемпинг), чтобы классифицировать место и отобразить соответствующие поля.

**Критерии приемки (Acceptance Criteria):**

**Дано (Given):**
- Пользователь открыл форму создания/редактирования места
- Форма содержит поле «Тип места»

**Когда (When):**
- Пользователь выбирает тип места:
  - «База отдыха»
  - «Дикое место»
  - «Кемпинг»

**Тогда (Then):**
- Выбранный тип сохраняется в поле `place_type` в базе данных
- В зависимости от типа отображаются или скрываются поля:
  - Для «База отдыха»: цена за день, макс. людей, удобства (все)
  - Для «Кемпинг»: макс. людей, удобства (только минимальные)
  - Для «Дикое место»: поля скрываются
- Тип места отображается в списке мест и на карте (иконка)

## Бизнес-правила

### BR1: Типы мест и их характеристики

**Правило:** Каждое место должно иметь один из трех типов:

| Тип | Описание | Обязательные поля | Дополнительные поля |
|-----|----------|------------------|-------------------|
| **База отдыха** | Организованное место с платным доступом и полным набором удобств | Название, описание, координаты, адрес, виды рыб, фото | Цена за день, макс. людей, удобства (все), сезонность, глубина, тип водоема, подход к воде, разрешение на рыбалку |
| **Дикое место** | Место для рыбалки с палаткой, без инфраструктуры | Название, описание, координаты, адрес, виды рыб, фото | Сезонность, глубина, тип водоема, подход к воде, разрешение на рыбалку |
| **Кемпинг** | Место для отдыха с палаткой с минимальной инфраструктурой | Название, описание, координаты, адрес, виды рыб, фото | Макс. людей, удобства (минимум), сезонность, глубина, тип водоема, подход к воде, разрешение на рыбалку |

**Логика:**
- При выборе типа отображаются/скрываются соответствующие поля
- Тип места нельзя изменить после создания места (future scope)
- Тип места влияет на отображение иконки на карте

---

### BR2: Обязательные поля для создания места

**Правило:** Для создания места обязательны следующие поля:

| Поле | Обязательное для всех типов | Валидация |
|------|---------------------------|-----------|
| Название | ✓ | 2-200 символов |
| Описание | ✓ | 10-5000 символов |
| Координаты (широта, долгота) | ✓ | Широта: -90 до 90, долгота: -180 до 180 |
| Адрес | ✓ | До 500 символов (может быть автозаполнено) |
| Тип места | ✓ | resort, wild_place, camping |
| Виды рыб | ✓ | Минимум 1 вид, максимум 10 видов |
| Фото | ✓ | Минимум 1 фото для публичных мест, до 5 фото |

---

### BR3: Условное отображение полей по типу места

**Правило:** Поля отображаются/скрываются в зависимости от выбранного типа места:

```
IF place_type = 'resort' THEN
  SHOW: price_per_day, max_people, facilities (полный список)
ELSE IF place_type = 'camping' THEN
  SHOW: max_people, facilities (минимальный список: туалет, душ, костровище)
ELSE IF place_type = 'wild_place' THEN
  HIDE: price_per_day, max_people, facilities
END IF
```

**Минимальный список удобств для кемпинга:**
- Туалет
- Костровище
- Душ (опционально)
- Парковка (опционально)

---

### BR4: Модерация публичных мест

**Правило:** Публичные места проходят модерацию перед публикацией:

```
IF is_public = true THEN
  SET status = 'pending_moderation'
  SEND notification to admins
  PLACE не отображается на публичной карте
  OWNER видит статус "На модерации"
END IF
```

**Статусы модерации:**
- `pending_moderation` — На модерации
- `active` — Одобрено (доступно всем)
- `rejected` — Отклонено (недоступно публично)

---

### BR5: Автоматическое заполнение адреса по координатам

**Правило:** При клике на карту выполняется обратное геокодирование:

```
IF user clicks on map THEN
  GET coordinates [lat, lng]
  CALL reverse geocoding API (Yandex Maps)
  PARSE response:
    - address: полное адрес
    - city: город
    - region: регион
  AUTOFILL form fields with parsed data
  IF address not found THEN
    SHOW warning "Адрес не определен, введите вручную"
    LEAVE fields empty
  END IF
END IF
```

**API для обратного геокодирования:**
```javascript
GET https://geocode-maps.yandex.ru/1.x/?apikey=API_KEY&format=json&geocode=lng,lat
```

---

### BR6: Отображение информации в баллоне

**Правило:** Информация в баллоне при наведении на точку ограничена:

```
BALLON content (hover):
  - title (название)
  - first image (если есть)
  - fish types (до 3 первых)
  - rating (если место публичное)
  - comment (первые 100 символов из описания)
END BALLON
```

**Примечание:** Полная информация отображается при клике на точку.

---

### BR7: Фильтрация и сортировка

**Правило:** Фильтры применяются с логикой OR для типов рыбы:

```
IF user selects fish_types = ['carp', 'bream'] THEN
  RETURN places WHERE fish_types INCLUDES 'carp' OR fish_types INCLUDES 'bream'
END IF
```

**Сортировка по умолчанию:** По дате создания (сначала новые)

## Ограничения и допущения

### Out of Scope (Вне рамок проекта)

Следующий функционал НЕ входит в текущую версию проекта:
1. **Поделиться местом** — кнопка UI будет подготовлена, но функционал не реализован
2. **Избранное** — кнопка UI будет подготовлена, но функционал не реализован
3. **Изменение типа места после создания** — тип можно выбрать только при создании
4. **Редактирование координат** — координаты можно указать только при создании
5. **Экспорт мест в GPX** — не планируется в текущей версии
6. **Автоматическое определение глубины водоема** — пользователь вводит вручную
7. **Интеграция с погодой** — не планируется в текущей версии
8. **Поделиться местом в социальных сетях** — не планируется в текущей версии
9. **Синхронизация с другими приложениями** — не планируется в текущей версии
