# Форма бизнес-задачи — Платформа FishMap

> Заполняется пользователем для постановки задачи конвейеру.
> Аналитик использует эту форму как входные данные.

---

## Шаблон

```markdown
# Бизнес-задача: [Название]

## Описание
[Что нужно сделать — 2-5 предложений]

## Тип задачи
- [ ] Новая фича
- [ ] Баг-фикс
- [ ] Улучшение (refactoring/optimization)
- [ ] Инфраструктура (Docker/CI/CD/мониторинг)

## Приоритет
- [ ] Critical — блокирует работу
- [ ] High — нужно в ближайшее время
- [ ] Medium — запланировано
- [ ] Low — когда будет время

## Для кого
- [ ] Все пользователи (гости)
- [ ] Зарегистрированные рыбаки
- [ ] Администраторы

## Ожидаемый результат
[Как выглядит успех — конкретный результат, который можно проверить]

## Дополнительный контекст
[Скриншоты, ссылки, примеры из других проектов]
```

---

## Примеры заполнения

### Пример 1: Новая фича

```markdown
# Бизнес-задача: Мои рыбные места

## Описание
Пользователи должны иметь возможность сохранять свои любимые рыбные места
в личный список. Каждое место можно добавить из карточки на карте или
из детальной страницы места. Список отображается в личном кабинете.

## Тип задачи
- [x] Новая фича

## Приоритет
- [x] High

## Для кого
- [x] Зарегистрированные рыбаки

## Ожидаемый результат
1. Кнопка "Сохранить" на карточке места и на странице места
2. Страница "Мои места" в личном кабинете со списком сохранённых мест
3. Удаление из избранного
4. Счётчик сохранений виден всем (анонимно)
```

### Пример 2: Баг-фикс

```markdown
# Бизнес-задача: Прогноз не отображается на мобильных

## Описание
На мобильных устройствах (iPhone/Android) страница прогноза клёва
показывает бесконечный спиннер. На десктопе всё работает.

## Тип задачи
- [x] Баг-фикс

## Приоритет
- [x] Critical

## Для кого
- [x] Все пользователи (гости)

## Ожидаемый результат
Прогноз клёва загружается и отображается на мобильных устройствах
в течение 3 секунд.
```

### Пример 3: Инфраструктура

```markdown
# Бизнес-задача: Настроить мониторинг Prometheus + Grafana

## Описание
Нужен мониторинг всех микросервисов: CPU, memory, request latency,
error rate. Дашборды для каждого сервиса. Алерты при падении.

## Тип задачи
- [x] Инфраструктура

## Приоритет
- [x] Medium

## Для кого
- [x] Администраторы

## Ожидаемый результат
1. Prometheus собирает метрики со всех сервисов
2. Grafana дашборд с общим обзором и детальными по сервисам
3. Alert при healthcheck fail любого сервиса
```

---

## Как конвейер обрабатывает форму

1. **Аналитик** получает форму → анализирует → определяет маршрут
2. Если задача архитектурно сложная → **Архитектор** → Аналитик
3. Если код/фича → **Разработчик** → **Тестировщик** → **DevOps**
4. Если инфраструктура → **DevOps** (прямая задача)

Пользователь может заполнить форму полностью или частично.
Аналитик задаст уточняющие вопросы через `question` tool если нужно.
