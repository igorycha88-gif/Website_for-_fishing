## 1. Цели и задачи

### 1.1 Бизнес-цель
Отображать глубины водоёмов поверх Яндекс Карт, чтобы рыбаки видели рельеф дна, выбирали места по глубине и получали автоматические рекомендации.

### 1.2 Пользовательская ценность
- Рыбак видит цветной оверлей глубин прямо на карте
- Клик по воде показывает точную глубину и подходящих рыб
- При добавлении места глубина определяется автоматически
- Прогноз клёва показывает рекомендуемую глубину для рыбы

### 1.3 Метрики успеха
- Время ответа `/api/v1/depth/point` < 50ms (при загруженном GeoTIFF)
- Тайл глубин отдаётся < 30ms (статические файлы через nginx)
- Слой глубин включается/выключается < 200ms

## 2. Функциональные требования

### 2.1 User Stories

**US-DEPTH-001: Включение слоя глубин**
**Как** рыбак, **я хочу** включить цветной оверлей глубин на карте, **чтобы** видеть рельеф дна водоёма.
- Given: пользователь на странице `/map`
- When: нажимает тумблер "Глубины" в панели слоёв
- Then: поверх базовой карты появляется полупрозрачный цветной оверлей (голубой градиент)
- And: отображается легенда цветов (0-2м, 2-5м, 5-10м, 10-20м, >20м)
- And: есть слайдер прозрачности (0-100%, по умолчанию 60%)

**US-DEPTH-002: Запрос глубины по клику**
**Как** рыбак, **я хочу** кликнуть по воде и узнать глубину, **чтобы** оценить подходит ли место для рыбы.
- Given: слой глубин включён (или выключен — запрос работает всегда)
- When: пользователь кликает по точке на воде
- Then: появляется popup с глубиной в метрах
- And: popup показывает координаты и точность данных
- And: popup показывает список рыб, подходящих для этой глубины
- And: есть кнопка "Добавить место" с предзаполненными координатами
- When: данных нет (мелкий водоём)
- Then: popup показывает "Глубина неизвестна" и предлагает ручной ввод

**US-DEPTH-003: Глубина в карточке места**
**Как** рыбак, **я хочу** видеть глубину в баллуне места, **чтобы** быстро оценить точку.
- Given: пользователь кликает на Placemark места
- When: открывается баллун
- Then: в баллуне отображается строка "Глубина: Xm" (если данные есть)
- And: глубина определяется автоматически по координатам места

**US-DEPTH-004: Автоглубина при добавлении места**
**Как** рыбак, **я хочу** чтобы глубина определилась автоматически при добавлении места, **чтобы** не вводить вручную.
- Given: пользователь кликает по карте для добавления места
- When: открывается форма AddPlaceForm
- Then: под координатами отображается бейдж "Глубина: Xm (авто)"
- And: есть кнопка "Указать вручную" для правки
- And: при сохранении глубота записывается в БД (приоритет ручного ввода)

**US-DEPTH-005: Рекомендуемая глубина в прогнозе**
**Как** рыбак, **я хочу** видеть рекомендуемую глубину для рыбы в прогнозе, **чтобы** знать на какой глубине ловить.
- Given: пользователь смотрит прогноз клёва
- When: открывается карточка рыбы
- Then: отображается "Рекомендуемая глубина: Xm-Ym"
- And: если известна глубина места — показывается сравнение (совпадение/несовпадение)

## 3. Нефункциональные требования

### 3.1 Производительность
- Point query: < 50ms (чтение из локального GeoTIFF через rasterio)
- Tile serving: < 30ms (статические файлы через nginx)
- Depth overlay: не более 256 тайлов одновременно в видимой области

### 3.2 Безопасность
- Endpoint `/api/v1/depth/point` — публичный (без авторизации)
- Endpoint `/api/v1/depth/tiles/*` — публичный (без авторизации)
- Rate limiting: 60 запросов/мин на IP для point query
- Валидация входных параметров (lat: -90..90, lon: -180..180)

### 3.3 Надёжность
- Если GeoTIFF не загружен — service здоров, но возвращает depth=null
- Service не падает при запросе за пределами данных
- structlog-логирование на каждом endpoint

## 4. Техническая архитектура

### 4.1 Изменения в БД

```sql
-- database/migrations/005_add_depth_to_places.sql
ALTER TABLE places ADD COLUMN IF NOT EXISTS depth NUMERIC(6,2);
ALTER TABLE places ADD COLUMN IF NOT EXISTS depth_source VARCHAR(10) DEFAULT 'auto';
-- depth_source: 'auto' (из GEBCO) | 'manual' (ручной ввод) | null (нет данных)
```

### 4.2 Новый микросервис: depth-service

| Параметр | Значение |
|----------|----------|
| Хост порт | 8008 |
| Контейнер порт | 8000 |
| Технология | FastAPI + rasterio |
| Источник данных | GEBCO GeoTIFF (Cloud Optimized GeoTIFF) |

**Endpoints:**
```
GET  /health                                    → healthcheck
GET  /api/v1/depth/point?lat=56.15&lon=37.05    → { depth, source, accuracy, fish_match }
GET  /api/v1/depth/tiles/{z}/{x}/{y}.png        → PNG растр с цветовой палитрой
```

### 4.3 API спецификация

**GET /api/v1/depth/point**
```json
// Response 200
{
  "depth": 4.2,           // глубина в метрах (отрицательная = над уровнем моря)
  "depth_display": "4.2 м",
  "source": "GEBCO_2024",
  "accuracy_m": 460,      // разрешение данных в метрах
  "has_data": true,       // false если данных нет
  "fish_match": [         // рыбы, подходящие для этой глубины
    { "name": "Щука", "icon": "🐟", "depth_range": "2-5 м" }
  ]
}
```

```json
// Response 200 (нет данных)
{
  "depth": null,
  "depth_display": null,
  "source": "GEBCO_2024",
  "has_data": false,
  "fish_match": []
}
```

## 7. Риски

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| GEBCO GeoTIFF не скачан | Высокая | Service работает без данных (depth=null), тайлы проксируются через GEBCO WMS |
| Внешний GEBCO WMS недоступен | Низкая | Fallback: пустые тайлы (прозрачные) |
| rasterio сложен в Docker build | Средняя | Использовать python:3.12-slim + GDAL из apt |
| Мелкие водоёмы без данных | Высокая | UX: "Глубина неизвестна" + ручной ввод |
