# ЧТЗ BUG-DEPTH-001: Слой «Глубины» не отображается на карте

**Тип:** Bug-fix (малая задача)
**Маршрут:** Аналитик → Разработчик → Тестировщик → DevOps → Тестировщик (постдеплой)
**Приоритет:** Высокий

---

## 1. Описание проблемы

При включении тумблера «Глубины» в панели слоёв на странице `/map` слой глубин не отображается на карте — визуально ничего не происходит.

## 2. Корневая причина

`frontend/components/YandexMap.tsx:537`:
```jsx
<Map instance={mapRef} ...>
```

Библиотека `@pbe/react-yandex-maps` v1.2.0 ожидает проп **`instanceRef`**, а не `instance`
(см. `node_modules/@pbe/react-yandex-maps/typings/util/typing.d.ts:9-10`).

Из-за неверного имени пропа `mapRef.current` **никогда не задаётся** (остаётся `null`).
Depth-эффект (`useEffect` строки 266-402) проверяет `mapRef.current` в строке 268-269 и
всегда выходит через `return` → слой, полигоны и метки глубин не отрисовываются.

## 3. Критерии приёмки

1. При включении тумблера «Глубины» на карте появляются полигоны водоёмов с цветовой заливкой по глубине
2. При выключении — слой и оверлеи корректно удаляются
3. Изменение прозрачности, цветовой схемы, меток и изобатов работает в реальном времени
4. Нет ошибок в консоли браузера при работе со слоем
5. Слой корректно работает после перезагрузки страницы

## 4. Файлы для изменения

| Файл | Изменение |
|------|-----------|
| `frontend/components/YandexMap.tsx` | `instance={mapRef}` → `instanceRef` callback + состояние `mapReady` в deps |
