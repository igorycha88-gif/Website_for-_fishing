## 1. Обзор

### 1.1. Описание функции

Система прогноза клева рыбы предоставляет пользователям персонализированные прогнозы активности рыбы на основе:
- Погодных условий (температура, давление, ветер, осадки)
- Времени года и суток
- Фазы луны
- Особенностей конкретных видов рыб
- **🆕 Сезонных зимних коэффициентов (градуированных по месяцам)**
- **🆕 Нерестовых периодов (запрет вылова)**
- Географического региона пользователя

### 1.2. Бизнес-ценность

**Для пользователей**:
- Планирование рыбалки в оптимальное время
- Выбор правильной стратегии ловли
- Экономия времени на неудачные выезды
- **🆕 Избежание штрафов за вылов в нерест**

**Для бизнеса**:
- Увеличение DAU/MAU (пользователи заходят за прогнозом)
- Дифференциация от конкурентов
- Удержание пользователей (retention)

---

## 2. Новое в v2.0

### 2.1. Градуированные зимние коэффициенты

Вместо бинарного подхода (1.2 или 0.3) используются дифференцированные коэффициенты:

| Месяц | Фаза | Активная рыба | Неактивная рыба |
|-------|------|---------------|-----------------|
| Декабрь | Первый лед | 0.9 | 0.2 |
| Январь | Глухозимье | 0.7 | 0.1 |
| Февраль | Последний лед | 1.0 | 0.15 |

### 2.2. Нерестовые периоды

Для каждого вида рыбы определены даты нереста:
- В период нереста bite_score = 0
- Отображается предупреждение "Нерестовый период — вылов запрещен"
- Пользователь информирован о законодательных ограничениях

## 5. Алгоритм прогнозирования v2.0

### 5.1. Градуированные зимние коэффициенты

```python
WINTER_MONTHLY_MULTIPLIERS = {
    12: {'active_fish': 0.9, 'inactive_fish': 0.2},   # Декабрь - первый лед
    1:  {'active_fish': 0.7, 'inactive_fish': 0.1},   # Январь - глухозимье
    2:  {'active_fish': 1.0, 'inactive_fish': 0.15},  # Февраль - последний лед
}

def get_season_multiplier(fish: FishSettings, month: int) -> float:
    if month in [12, 1, 2]:
        multipliers = WINTER_MONTHLY_MULTIPLIERS[month]
        return multipliers['active_fish'] if fish.active_in_winter else multipliers['inactive_fish']
    elif month in [3, 4, 5]:  # Весна
        return 1.0
    elif month in [6, 7, 8]:  # Лето
        return 1.0
    else:  # Осень
        return 1.0
```

### 5.2. Проверка нерестового периода

```python
def is_in_spawn_period(fish: FishSettings, date: date) -> tuple[bool, str]:
    """
    Проверяет, находится ли указанная дата в нерестовом периоде рыбы.
    
    Returns:
        tuple: (is_spawn: bool, message: str)
    """
    if fish.spawn_start_month is None or fish.spawn_end_month is None:
        return False, ""
    
    month = date.month
    day = date.day
    
    start_month = fish.spawn_start_month
    end_month = fish.spawn_end_month
    
    is_spawn = False
    
    if start_month <= end_month:
        # Нерест в пределах одного года (например, апрель-май)
        if start_month == end_month:
            is_spawn = start_month == month and fish.spawn_start_day <= day <= fish.spawn_end_day
        else:
            is_spawn = (
                (start_month == month and day >= fish.spawn_start_day) or
                (end_month == month and day <= fish.spawn_end_day) or
                (start_month < month < end_month)
            )
    else:
        # Нерест переходит через год (например, декабрь-февраль, как у налима)
        is_spawn = (
            (month >= start_month) or
            (month <= end_month)
        )
    
    if is_spawn:
        month_names = {
            1: 'января', 2: 'февраля', 3: 'марта', 4: 'апреля',
            5: 'мая', 6: 'июня', 7: 'июля', 8: 'августа',
            9: 'сентября', 10: 'октября', 11: 'ноября', 12: 'декабря'
        }
        message = f"Нерестовый период ({fish.spawn_start_day} {month_names[start_month]} - {fish.spawn_end_day} {month_names[end_month]}) — вылов запрещен"
        return True, message
    
    return False, ""
```

### 5.3. Обновленная формула расчета

```python
def calculate_bite_score(
    weather: WeatherConditions,
    fish: FishSettings,
    hour: int,
    month: int,
    date: date
) -> dict:
    # Проверка нереста
    is_spawn, spawn_message = is_in_spawn_period(fish, date)
    if is_spawn:
        return {
            'bite_score': 0,
            'is_spawn_period': True,
            'spawn_message': spawn_message,
            'time_of_day': get_time_of_day(hour).value,
            'components': {},
            'season_multiplier': 0,
        }
    
    time_of_day = get_time_of_day(hour, weather.sunrise, weather.sunset)
    
    # Расчет компонентов
    temp_score = calculate_temperature_score(weather, fish)
    pressure_score = calculate_pressure_score(weather, fish)
    wind_score = calculate_wind_score(weather, fish)
    moon_score = calculate_moon_score(weather, fish)
    precip_score = calculate_precipitation_score(weather)
    time_score = calculate_time_score(time_of_day, fish)
    
    # Взвешенная сумма
    base_score = (
        temp_score * 0.25 +
        pressure_score * 0.25 +
        time_score * 0.20 +
        wind_score * 0.15 +
        moon_score * 0.10 +
        precip_score * 0.05
    )
    
    # 🆕 Градуированный сезонный множитель
    season_mult = get_season_multiplier(fish, month)
    bite_score = base_score * season_mult
    
    # Ограничение 0-100
    bite_score = max(0, min(100, bite_score))
    
    return {
        'bite_score': round(bite_score, 1),
        'is_spawn_period': False,
        'spawn_message': None,
        'time_of_day': time_of_day.value,
        'components': {
            'temperature': round(temp_score, 1),
            'pressure': round(pressure_score, 1),
            'wind': round(wind_score, 1),
            'moon': round(moon_score, 1),
            'precipitation': round(precip_score, 1),
            'time_of_day': round(time_score, 1),
        },
        'season_multiplier': season_mult,
    }
```

## 9. Non-Functional Requirements

### 9.1. Performance
- **API Response**: < 200ms для GET запросов прогноза
- **Scheduled Task**: сбор данных для 85 регионов < 5 минут
- **Database**: индексы на всех частых запросах
- **🆕 Spawn Check**: < 1ms (простая проверка дат)

### 9.2. Reliability
- **Fallback API**: Open-Meteo как резерв
- **Retry Logic**: 3 попытки при ошибке API
- **Data Retention**: погодные данные хранятся 30 дней
- **🆕 Backward Compatibility**: рыбы без spawn периодов работают как раньше

### 9.3. Caching
- **Redis**: кэширование прогнозов на 1 час
- **Cache Key**: `forecast:{region_id}:{date}:{fish_type_id}`
