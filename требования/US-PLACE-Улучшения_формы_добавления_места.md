## Обзор изменений

## US-PLACE-001: Визуализация выбранной точки на карте

**As a** зарегистрированный пользователь,
**I want to** видеть визуальное подтверждение выбранной точки на карте при добавлении места,
**So that** я могу быть уверен, что корректно установил место на карте.

**Priority**: Medium

**Actors**:
- [x] Зарегистрированный пользователь
- [x] Moderator
- [x] Admin

**Acceptance Criteria**:

**AC1: Отображение выбранной точки на карте**
- **Given** пользователь авторизован
- **And** находится на вкладке "Мои места"
- **When** кликает на карту в точке с координатами 55.75, 37.61
- **Then** на карте появляется временная метка (маркер) в выбранной точке
- **And** маркер имеет синий цвет (#3b82f6)
- **And** маркер отличается от уже существующих мест на карте

**AC2: Маркер исчезает после закрытия формы**
- **Given** пользователь кликнул на карту
- **And** появился временный маркер
- **And** открылась форма добавления места
- **When** пользователь закрывает форму без сохранения
- **Then** временный маркер исчезает с карты

**AC3: Маркер обновляется при повторном клике**
- **Given** пользователь кликнул на карту в точке A
- **And** появился временный маркер
- **When** пользователь кликает на карту в точке B
- **Then** временный маркер перемещается в точку B
- **And** старое местоположение маркера удаляется

**Non-Functional Requirements**:
- **Performance**: Маркер должен появляться мгновенно (<100ms)
- **Visual**: Маркер должен быть хорошо видим на карте

**Dependencies**:
- Зависит от: YandexMap компонента
- Блокирует: Нет

**Definition of Done**:
- [ ] Временный маркер с синим цветом добавлен на карту при клике
- [ ] Маркер удаляется при закрытии формы
- [ ] Маркер перемещается при повторном клике на карту
- [ ] CSS стили для маркера определены
- [ ] Тесты написаны

## US-PLACE-002: Добавление поля "Тип водоема"

**As a** зарегистрированный пользователь,
**I want to** указывать тип водоема при добавлении места,
**So that** другие пользователи могли быстро понимать, где планируется рыбалка.

**Priority**: High

**Actors**:
- [x] Зарегистрированный пользователь
- [x] Moderator
- [x] Admin

**Acceptance Criteria**:

**AC1: Отображение поля "Тип водоема" в форме**
- **Given** пользователь авторизован
- **And** открывает форму добавления места
- **Then** видит поле "Тип водоема" с обязательной отметкой (*)
- **And** в выпадающем списке доступны варианты: "Река", "Озеро", "Море"
- **And** поле расположено логично в форме (после "Тип места")

**AC2: Выбор типа водоема**
- **Given** пользователь открывает форму добавления места
- **When** выбирает тип водоема "Река"
- **And** заполняет остальные обязательные поля
- **And** нажимает "Сохранить"
- **Then** место успешно сохраняется
- **And** в записи места указан тип водоема "Река"

**AC3: Валидация обязательности поля**
- **Given** пользователь открывает форму добавления места
- **When** не выбирает тип водоема
- **And** пытается сохранить форму
- **Then** видит ошибку "Выберите тип водоема"
- **And** форма не отправляется

**Non-Functional Requirements**:
- **Data Integrity**: Тип водоема хранится в базе данных
- **Performance**: Валидация происходит мгновенно (<100ms)

**Dependencies**:
- Зависит от: Обновления типов Place в БД
- Блокирует: Нет

**Definition of Done**:
- [ ] Поле "water_type" добавлено в типы TypeScript (Place, PlaceCreate, PlaceUpdate)
- [ ] Поле добавлено в форму AddPlaceForm
- [ ] Валидация обязательности реализована
- [ ] Migration для добавления колонки в таблицу places создана
- [ ] Unit тесты написаны
- [ ] UI тесты написаны

## US-PLACE-003: Защита координат от редактирования

**As a** зарегистрированный пользователь,
**I want to** чтобы координаты (ширина и долгота) заполнялись автоматически и не редактировались вручную,
**So that** я не могу случайно ввести неверные координаты и получить место в неправильном месте на карте.

**Priority**: High

**Actors**:
- [x] Зарегистрированный пользователь
- [x] Moderator
- [x] Admin

**Acceptance Criteria**:

**AC1: Поля координат не редактируемы**
- **Given** пользователь открывает форму добавления места
- **When** смотрит на поля "Широта" и "Долгота"
- **Then** поля отображают значения автоматически заполненных координат
- **And** поля недоступны для редактирования (disabled)

**AC2: Автозаполнение координат из карты**
- **Given** пользователь открывает вкладку "Мои места"
- **When** кликает на карту в точке с координатами 55.75, 37.61
- **And** открывается форма добавления места
- **Then** поле "Широта" содержит значение 55.75
- **And** поле "Долгота" содержит значение 37.61

**AC3: Открытие формы без выбора точки на карте**
- **Given** пользователь нажимает кнопку "Добавить место" без клика по карте
- **When** открывается форма добавления места
- **Then** поле "Широта" содержит значение по умолчанию (55.7558)
- **And** поле "Долгота" содержит значение по умолчанию (37.6173)
- **And** поля недоступны для редактирования
- **And** пользователь видит подсказку "Выберите точку на карте для установки координат"

**Non-Functional Requirements**:
- **UX**: Поля визуально отличаются от редактируемых полей
- **Accessibility**: Поля отмечены атрибутом disabled для screen readers

**Dependencies**:
- Зависит от: YandexMap компонента
- Блокирует: Нет

**Definition of Done**:
- [ ] Поля "latitude" и "longitude" в форме AddPlaceForm имеют атрибут disabled
- [ ] Автозаполнение координат при клике на карту работает
- [ ] Значения по умолчанию заданы
- [ ] Визуальные стили для disabled полей применены
- [ ] Unit тесты написаны
- [ ] Тесты на автозаполнение написаны

## US-PLACE-004: Автозаполнение адреса из координат

**As a** зарегистрированный пользователь,
**I want to** чтобы адрес автоматически заполнялся на основе выбранной точки на карте,
**So that** мне не нужно вручную вводить адрес, что экономит время и предотвращает ошибки.

**Priority**: High

**Actors**:
- [x] Зарегистрированный пользователь
- [x] Moderator
- [x] Admin

**Acceptance Criteria**:

**AC1: Автозаполнение адреса при клике на карту**
- **Given** пользователь открывает вкладку "Мои места"
- **When** кликает на карту в точке с координатами 55.75, 37.61
- **And** Яндекс Карты могут определить адрес для этой точки
- **Then** в форме добавления места поле "Адрес" автоматически заполняется адресом
- **Example**: "г. Москва, ул. Примерная, д. 1"

**AC2: Обработка случая, когда адрес не найден**
- **Given** пользователь кликает на карту в точке, где Яндекс Карты не могут определить адрес
- **When** открывается форма добавления места
- **Then** поле "Адрес" остается пустым
- **And** пользователь видит подсказку "Адрес не найден. Введите вручную."
- **And** пользователь может ввести адрес вручную

**AC3: Возможность редактирования адреса**
- **Given** пользователь кликнул на карту
- **And** адрес автоматически заполнился как "г. Москва, ул. Примерная"
- **And** открывается форма добавления места
- **When** пользователь редактирует адрес на "г. Москва, ул. Другая, д. 2"
- **And** нажимает "Сохранить"
- **Then** место сохраняется с измененным адресом

**Non-Functional Requirements**:
- **Performance**: Геокодинг должен выполняться быстро (<500ms)
- **Error Handling**: Graceful degradation, если геокодинг недоступен

**Dependencies**:
- Зависит от: Яндекс Карты API Geocoder
- Блокирует: Нет

**Definition of Done**:
- [ ] Геокодинг через Яндекс Карты API реализован
- [ ] Автозаполнение поля "address" работает при клике на карту
- [ ] Обработка ошибки, когда адрес не найдена
- [ ] Поле "address" остается редактируемым
- [ ] Показ подсказки пользователю
- [ ] Unit тесты написаны
- [ ] Интеграционные тесты с Яндекс Карты API

## US-PLACE-005: Сделать поле с фотографиями не обязательным

**As a** зарегистрированный пользователь,
**I want to** иметь возможность добавлять место без фотографий,
**So that** я могу быстро сохранить информацию о месте, даже если у меня нет фото под рукой.

**Priority**: Medium

**Actors**:
- [x] Зарегистрированный пользователь
- [x] Moderator
- [x] Admin

**Acceptance Criteria**:

**AC1: Форма отправляется без фотографий**
- **Given** пользователь открывает форму добавления места
- **When** заполняет все обязательные поля (название, координаты, адрес, тип места, тип водоема, тип подъезда, виды рыбы)
- **And** НЕ загружает ни одной фотографии
- **And** нажимает "Сохранить"
- **Then** место успешно сохраняется
- **And** в записи места массив images пустой

**AC2: Отображение мест без фото в списке**
- **Given** пользователь имеет место без фотографий
- **When** открывает вкладку "Мои места"
- **Then** в списке мест для места без фото отображается плейсхолдер
- **And** плейсхолдер содержит иконку типа места (дикий, кэмпинг, база отдыха)

**AC3: Плейсхолдеры для разных типов мест**
- **Given** пользователь имеет три места без фото: дикое, кэмпинг, база отдыха
- **When** просматривает список мест
- **Then** для дикого места показана иконка "🌲"
- **And** для кэмпинга показана иконка "⛺"
- **And** для базы отдыха показана иконка "🏨"

**AC4: Отображение места без фото в модальном окне**
- **Given** пользователь кликает на место без фотографий
- **When** открывается модальное окно с детальной информацией
- **Then** не показывается блок с изображениями
- **And** отображается только описание и другая информация

**Non-Functional Requirements**:
- **UX**: Плейсхолдер визуально привлекательный и понятный
- **Performance**: Отсутствие проверки на обязательность фото ускоряет сохранение

**Dependencies**:
- Зависит от: Компонента списка мест и модального окна
- Блокирует: Нет

**Definition of Done**:
- [ ] Валидация на обязательность фото удалена из формы AddPlaceForm
- [ ] Плейсхолдер с иконкой типа места добавлен в список мест (MyPlacesTab)
- [ ] Плейсхолдер с иконкой типа места добавлен в модальное окно
- [ ] Unit тесты написаны
- [ ] UI тесты написаны
- [ ] Проверка во всех местах, где отображаются фото

## Database Schema Change

**Service**: places-service
**Date**: 2024-02-12
**Author**: Business/System Analyst
**Reason**: Добавление поля "Тип водоема" для улучшения классификации мест

### Changes

#### Alter Table: places

```sql
-- Add new column water_type
ALTER TABLE places
ADD COLUMN water_type VARCHAR(20) NOT NULL DEFAULT 'lake';

-- Add check constraint for water_type values
ALTER TABLE places
ADD CONSTRAINT chk_water_type_valid
CHECK (water_type IN ('river', 'lake', 'sea'));

-- Add comment
COMMENT ON COLUMN places.water_type IS 'Тип водоема: river - река, lake - озеро, sea - море';

-- Create index for filtering by water_type
CREATE INDEX idx_places_water_type ON places(water_type);
```

### Migration Script

```sql
-- Migration: add_water_type_to_places
-- Date: 2024-02-12

BEGIN;

-- Apply changes here...

```


### Impact Analysis

## API Specification Changes

### Places API - Create Place Endpoint

#### POST /api/v1/places/my

**Request Body Changes**:

```json
{
  "name": "Озеро Рыбное",
  "description": "Хорошее место для рыбалки",
  "latitude": 55.7558,
  "longitude": 37.6173,
  "address": "г. Москва, ул. Примерная",
  "place_type": "wild",
  "access_type": "car",
  "fish_types": ["1", "2"],
  "seasonality": ["spring", "summer"],
  "visibility": "private",
  "images": [],
  "water_type": "lake"
}
```

**New Field**:
- `water_type` (string, required) - Тип водоема
  - `river` - Река
  - `lake` - Озеро
  - `sea` - Море

**Changes in Validation**:
- `images` changed from required to optional
- `water_type` added as required field with default value "lake"

### Places API - Update Place Endpoint

#### PUT /api/v1/places/my/{id}

**Request Body Changes**:

```json
{
  "name": "Новое название",
  "water_type": "river",
  "images": []
}
```

**New Field**:
- `water_type` (string, optional) - Тип водоема

## Risk Analysis

### Risk 1: Геокодинг Яндекс Карт недоступен

**Category**: Technical
**Probability**: Low
**Impact**: Medium

**Description**: Яндекс Карты API может быть временно недоступен или вернуть ошибку при попытке определить адрес по координатам.

**Potential Impact**:
1. Пользователи не смогут получить автоматический адрес
2. Ухудшение UX

**Mitigation Strategies**:
1. **Prevent**: Использовать fallback на ручной ввод адреса
2. **Mitigate**: Показывать понятное сообщение об ошибке
3. **Accept**: Пользователь всегда может ввести адрес вручную

**Owner**: Frontend Developer
**Review Date**: 2024-02-19

### Risk 2: Нарушение обратной совместимости

**Category**: Technical
**Probability**: Low
**Impact**: Medium

**Description**: Добавление обязательного поля water_type может нарушить совместимость с существующими данными или клиентскими приложениями.

**Potential Impact**:
1. Ошибки при создании мест старыми версиями frontend
2. Ошибки при чтении данных

**Mitigation Strategies**:
1. **Prevent**: Установить значение по умолчанию "lake"
2. **Mitigate**: Использовать миграцию с NOT NULL DEFAULT
3. **Accept**: Версионирование API для обратной совместимости

**Owner**: Backend Developer
**Review Date**: 2024-02-19
