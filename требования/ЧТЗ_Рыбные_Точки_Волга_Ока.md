# ЧТЗ: Рыбные точки на Волге и Оке (демо-данные + слой на карте)

> **ID задачи:** US-CATCH-MAP-001
> **Маршрут:** Маршрут 2 (сложная) — Аналитик → Архитектор → Аналитик → Разработчик → Тестировщик → DevOps → Тестировщик (постдеплой)
> **Исполнитель после ЧТЗ:** Разработчик (fullstack)
> **Связанный ADR:** `требования/ADR_Рыбные_Точки_Волга_Ока.md`

---

## 1. Постановка задачи

Пользователь хочет видеть на карте (на главной и в личном кабинете) точки, где ловили рыбу на реках Волга и Ока. Каждая точка — значок рыбы; при нажатии отображается, какую рыбу ловили в данной точке (вид, сезон, глубина, снасть/наживка).

### Решение по данным (выбор пользователя)
Подход: **демо-данные по известным рыболовным местам** Волги и Оки. Данные честно помечаются как демонстрационные (`is_demo = true`). В будущем точки могут дополняться реальными пользовательскими уловами.

> Скрапинг закрытых сервисов (Fishbrain, «Навигатор рыболова» и т.п.) НЕ выполняется — нарушает их правила и авторские права.

## 2. Область изменений (scope)

### Backend (places-service)
- Новая таблица `catch_points` (см. ADR).
- Новый контекст endpoints `GET /api/v1/catches`, `GET /api/v1/catches/{id}`.
- Seed демо-данных: ~30–40 точек по Волге и Оке (Тверь, Калязинский плёс, Рыбинское вдхр., Ярославль, Кострома, Нижний Новгород, Чебоксары, Казань; Ока — Калуга, Серпухов, Коломна, Рязань, Муром, Нижний Новгород-слияние).
- structlog-логирование на каждом endpoint.

### Frontend
- Тип `CatchPoint`, хук `useCatches`, точка в `API_ENDPOINTS`.
- Слой рыбных точек в `YandexMap.tsx`: иконка по виду рыбы (emoji + цвет), кликабельный балун.
- Главная страница (`app/page.tsx`): новая секция с интерактивной картой + рыбные точки Волги/Оки.
- Карта (`app/map/page.tsx`): рыбные точки поверх мест.
- Личный кабинет (`profile/components/MyPlacesTab.tsx` или новая вкладка): карта + мои уловы.

### Тесты
- pytest (backend), Jest (frontend unit), Playwright E2E + UI.

## 3. Критерии приёмки

1. `GET /api/v1/catches` возвращает список рыбных точек с координатами, видом рыбы, сезоном, глубиной. Статус 200.
2. Фильтрация по `river` (volga/oka), `fish_type_id`, bbox (`min_lat`, `min_lon`, `max_lat`, `max_lon`).
3. `GET /api/v1/catches/{id}` возвращает 200 для существующей, 404 для отсутствующей.
4. Демо-данные Волги и Оки присутствуют после запуска seed.
5. На карте главные точки отображаются значком рыбы (emoji по виду), кликабельны, в балуне — вид рыбы, место, сезон, глубина, наживка/снасть, пометка «демо-данные».
6. Рыбные точки видны на: главной странице (новая секция), странице `/map`, в ЛК.
7. structlog-логи на endpoints (request_started, request_completed, request_failed).
8. pytest/ruff/ESLint/tsc — зелёные.
9. Playwright E2E + UI тесты проходят.
10. После деплоя — healthcheck всех сервисов OK, точки доступны на живом стенде.

## 6. Риски

- **Точность демо-координат** — mitigated: общедоступная география рек, пометка `is_demo`.
- **Производительность слоя** — mitigated: bbox-фильтрация на бэке, clusterer на фронте.
