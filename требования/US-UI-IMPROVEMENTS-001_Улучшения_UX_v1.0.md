## 1. Обзор

### 1.1. Описание изменений

Документ описывает три независимых улучшения UX:

1. **Редизайн вкладки "Мои места"** — перенос карточек под карту, расширение карты
2. **Прогноз клева** — исправление кнопки "Показать все N" для раскрытия полного списка рыб
3. **Рекомендации по ловле** — сезонные рекомендации с разделением на мирную рыбу и хищника

### 1.2. Бизнес-ценность

| Улучшение | Ценность |
|-----------|----------|
| Редизайн "Мои места" | Улучшение UX, больше места для карты, удобный просмотр списка |
| Прогноз клева | Пользователь видит прогноз по ВСЕМ рыбам региона |
| Рекомендации | Персонализированные советы по снастям, учет сезона и типа рыбы |

## 2. User Story 1: Редизайн вкладки "Мои места"

### US-1-1: Карта на всю ширину

**As a** зарегистрированный пользователь,
**I want to** видеть карту на всю ширину страницы,
**So that** я могу лучше ориентироваться по местам и видеть больше территории.

**Priority**: High

**Acceptance Criteria**:

**AC1: Карта занимает полную ширину**
- **Given** пользователь на вкладке "Мои места"
- **When** страница загружена
- **Then** карта занимает всю ширину контейнера (col-span-full или 100%)
- **And** высота карты увеличена (например, 400-450px)

**AC2: Карточки мест под картой**
- **Given** карта отображается сверху
- **When** пользователь прокручивает вниз
- **Then** видит блок "Мои места" со списком карточек под картой
- **And** карточки располагаются в колонку (grid-cols-1 или flex flex-col)

---

### US-1-2: Компактные информативные карточки

**As a** зарегистрированный пользователь,
**I want to** видеть информативные карточки мест, которые можно развернуть,
**So that** я могу быстро найти нужное место и при желании увидеть детали.

**Priority**: High

**Acceptance Criteria**:

**AC1: Компактная карточка в свернутом состоянии**
- **Given** список мест отображается под картой
- **When** карточка в свернутом виде
- **Then** отображается:
  - Фото (60x60px, скругленное)
  - Название места (жирный шрифт)
  - Адрес (серый, truncate)
  - Теги: тип места, видимость (private/public), тип подъезда
  - Иконки видов рыб (до 3 иконок, далее "+N")
  - Иконка для развертывания (ChevronDown)

**AC2: Развернутая карточка**
- **Given** пользователь кликает на карточку
- **When** карточка разворачивается
- **Then** отображается дополнительная информация:
  - Описание (если есть)
  - Сезонность
  - Координаты
  - Все фото (если >1)
  - Кнопки: "Показать на карте", "Редактировать", "Удалить"

**AC3: Клик на свернутую карточку**
- **Given** карточка свернута
- **When** пользователь кликает на неё
- **Then** карточка разворачивается с анимацией (framer-motion)
- **And** карта центрируется на этом месте (опционально)

**AC4: Клик на развернутую карточку**
- **Given** карточка развернута
- **When** пользователь кликает вне карточки или на иконку свернуть
- **Then** карточка сворачивается

---

### US-1-3: Сворачиваемый блок списка мест

**As a** зарегистрированный пользователь,
**I want to** иметь возможность свернуть весь блок со списком мест,
**So that** я мог видеть только карту, если мне не нужен список.

**Priority**: Medium

**Acceptance Criteria**:

**AC1: Кнопка сворачивания блока**
- **Given** блок "Мои места" отображается под картой
- **When** пользователь видит заголовок блока
- **Then** справа от заголовка есть кнопка-иконка (ChevronUp/Down)
- **And** при клике блок сворачивается/разворачивается

**AC2: Состояние блока сохраняется**
- **Given** пользователь свернул блок списка
- **When** перезагружает страницу
- **Then** состояние сохраняется в localStorage (опционально)

## 3. User Story 2: Прогноз клева — "Показать все"

### US-2-1: Раскрытие полного списка рыб

**As a** пользователь,
**I want to** видеть прогноз клева по всем рыбам региона при нажатии "Показать все",
**So that** я могу выбрать рыбу для ловли не только из топ-4.

**Priority**: High

**Acceptance Criteria**:

**AC1: Кнопка "Показать все N" отображает все рыбы**
- **Given** прогноз клева загружен
- **And** в регионе доступно N рыб (N > 4)
- **When** пользователь нажимает "Показать все N"
- **Then** отображается полный список всех рыб региона
- **And** кнопка меняется на "Свернуть"
- **And** список разворачивается с анимацией

**AC2: Сортировка полного списка**
- **Given** отображается полный список рыб
- **When** пользователь просматривает список
- **Then** рыбы отсортированы по убыванию bite_score

**AC3: Кнопка "Свернуть"**
- **Given** отображается полный список
- **When** пользователь нажимает "Свернуть"
- **Then** отображается только топ-4 рыбы
- **And** кнопка снова показывает "Показать все N"

**AC4: Сохранение развернутой рыбы**
- **Given** пользователь раскрыл конкретную рыбу (expandedFish)
- **When** нажимает "Показать все"
- **Then** развернутая рыба остается развернутой
- **And** остальные рыбы отображаются свернутыми

---

### Техническое решение

**Проблема**: В текущем коде (FishingForecast.tsx:458-465) кнопка "Показать все" устанавливает `expandedFish = "all"`, но не отображает дополнительные рыбы.

**Решение**:

```tsx
const [showAllFish, setShowAllFish] = useState(false);

const getDisplayedFish = (): FishForecast[] => {
  if (!forecast || !forecast.forecasts) return [];
  const sorted = [...forecast.forecasts].sort((a, b) => getAverageScore(b) - getAverageScore(a));
  return showAllFish ? sorted : sorted.slice(0, 4);
};

// В рендере:
{getDisplayedFish().map((fishForecast) => (
  // ... карточка рыбы
))}

{forecast.forecasts && forecast.forecasts.length > 4 && (
  <button
    onClick={() => setShowAllFish(!showAllFish)}
    className="w-full mt-3 py-2 text-blue-600 hover:text-blue-700 text-sm font-medium"
  >
    {showAllFish ? "Свернуть" : `Показать все (${forecast.forecasts.length})`}
  </button>
)}
```

## 4. User Story 3: Рекомендации по ловле

### US-3-1: Сезонные рекомендации

**As a** пользователь,
**I want to** получать рекомендации по ловле, учитывающие сезон,
**So that** я не получаю невозможные советы (например, "проводка" зимой подо льдом).

**Priority**: High

**Acceptance Criteria**:

**AC1: Определение текущего сезона**
- **Given** система генерирует рекомендацию
- **When** определяется сезон
- **Then** используются календарные сезоны:
  - Зима: декабрь, январь, февраль
  - Весна: март, апрель, май
  - Лето: июнь, июль, август
  - Осень: сентябрь, октябрь, ноябрь

**AC2: Зимние рекомендации**
- **Given** текущий сезон — Зима
- **When** генерируется рекомендация
- **Then** НЕ используются термины: "проводка", "заброс", "спиннинг"
- **And** используются термины: "мормышка", "блеснение", "поплавок", "жерлица", "лунка"

**AC3: Весенние/летние/осенние рекомендации**
- **Given** сезон НЕ Зима
- **When** генерируется рекомендация
- **Then** доступны все типы ловли: спиннинг, фидер, поплавок, нахлыст

---

### US-3-2: Рекомендации для мирной рыбы

**As a** пользователь,
**I want to** получать рекомендации по наживкам для мирной рыбы,
**So that** я знаю чем прикармливать и на что ловить.

**Priority**: High

**Acceptance Criteria**:

**AC1: Мирная рыба определяется по категории**
- **Given** рыба из категории `peaceful` или `commercial`
- **When** генерируется рекомендация
- **Then** рекомендуются наживки (натуральные насадки)

**AC2: Список наживок по сезонам**

| Сезон | Наживки для мирной рыбы |
|-------|------------------------|
| Зима | Мотыль, опарыш, червь, тесто, манная болтушка |
| Весна | Червь, опарыш, мотыль, перловка, кукуруза, горох |
| Лето | Кукуруза, горох, перловка, червь, опарыш, тесто, бойлы, хлеб |
| Осень | Червь, опарыш, мотыль, перловка, кукуруза |

---

### US-3-3: Рекомендации для хищника

**As a** пользователь,
**I want to** получать рекомендации по приманкам для хищной рыбы,
**So that** я знаю какие блесны/воблеры/силикон использовать.

**Priority**: High

**Acceptance Criteria**:

**AC1: Хищник определяется по категории**
- **Given** рыба из категории `predatory` или `sport`
- **When** генерируется рекомендация
- **Then** рекомендуются приманки (искусственные)

**AC2: Список приманок по сезонам**

| Сезон | Приманки для хищника |
|-------|---------------------|
| Зима | Блесны вертикальные, балансиры, мормышки крупные, жерлицы |
| Весна | Воблеры (суспендеры), джиг, блесны колеблющиеся |
| Лето | Воблеры, силикон (джиг-головки), вращающиеся блесны, воблеры попперы |
| Осень | Джиг (крупный силикон), воблеры (глубоководные), блесны |

## 5. Non-Functional Requirements

### 5.1. Performance
- **Карта**: загрузка < 2 сек
- **Список карточек**: рендеринг < 100ms для 20 карточек
- **Прогноз**: API < 200ms
- **Анимации**: 60fps, длительность 200-300ms

### 5.2. UX
- **Responsive**: адаптация под мобильные устройства
- **Accessibility**: поддержка screen readers
- **Дизайн**: соответствие текущему стилю (Tailwind, primary-sea цвета)
