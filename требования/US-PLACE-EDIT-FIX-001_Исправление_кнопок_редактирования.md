# Требования: Исправление отсутствия кнопок редактирования/удаления в модальном окне места

**ID**: US-PLACE-EDIT-FIX-001
**Version**: 1.0
**Priority**: High (Critical Bug)
**Status**: Draft
**Date**: 2024-02-17
**Related**: US-PLACE-EDIT-001

---

## 1. Problem Statement

### 1.1 Описание проблемы
При клике на карточку места в разделе "Мои места" (/profile → Мои места) открывается модальное окно просмотра, но кнопки "Редактировать" и "Удалить" не отображаются.

### 1.2 Ожидаемое поведение (согласно US-PLACE-EDIT-001)
- Модальное окно просмотра содержит кнопку "Редактировать" (синяя/primary-sea)
- Модальное окно просмотра содержит кнопку "Удалить" (красная)
- Кнопки также есть в заголовке модального окна (иконки Pencil и Trash2)

### 1.3 Фактическое поведение
- Модальное окно открывается
- Информация о месте отображается корректно
- Кнопки редактирования и удаления отсутствуют

---

## 2. Root Cause Analysis

### 2.1 Код-ревал компонентов

| Файл | Строки | Код | Статус |
|------|--------|-----|--------|
| `MyPlacesTab.tsx` | 449-608 | Модальное окно с кнопками | ✅ Присутствует в коде |
| `MyPlacesTab.tsx` | 459-480 | Кнопки в заголовке (Pencil, Trash2, X) | ✅ Присутствует в коде |
| `MyPlacesTab.tsx` | 589-603 | Кнопки внизу (Редактировать, Удалить) | ✅ Присутствует в коде |
| `EditPlaceForm.tsx` | 1-447 | Компонент формы редактирования | ✅ Существует |
| `ConfirmDialog.tsx` | 1-85 | Компонент диалога подтверждения | ✅ Существует |
| `tailwind.config.ts` | 14-15 | primary-sea цвет | ✅ Определён |

### 2.2 Условие рендеринга модального окна

```tsx
// MyPlacesTab.tsx:449
{selectedPlace && !showEditForm && !showDeleteConfirm && (
  // ... модальное окно с кнопками
)}
```

**Условие корректно** - модальное окно рендерится когда:
- `selectedPlace` не null
- `showEditForm` = false
- `showDeleteConfirm` = false

### 2.3 Возможные причины

#### Причина 1: Старая версия в браузере (ВЕРОЯТНО - 80%)
- Код на диске обновлён, но браузер кэшировал старую версию
- Next.js не пересобрал страницу
- Service Worker отдаёт старую версию

#### Причина 2: Неправильная страница (ВОЗМОЖНО - 15%)
- Пользователь находится на `/map` (публичная карта), а не на `/profile` → "Мои места"
- Страница `/map` НЕ передаёт `onPlaceClick` в YandexMap
- На `/map` клик на маркер не открывает модальное окно

#### Причина 3: CSS/Стили (МАЛО ВЕРОЯТНО - 5%)
- Кнопки рендерятся, но скрыты CSS
- Проблема с z-index
- Tailwind классы не скомпилированы

### 4.3 Если кнопки есть в DOM но не видны (Решение В)

**Проблема с CSS классом `primary-sea`:**

Заменить кнопки на явные стили:

```tsx
// В заголовке (lines 460-473)
<button
  onClick={() => setShowEditForm(true)}
  className="p-2 hover:bg-blue-50 rounded-lg transition-colors text-blue-600"
  style={{ backgroundColor: 'transparent', color: '#00b4d8' }}
  title="Редактировать"
>
  <Pencil className="w-5 h-5" />
</button>

// Внизу (lines 589-603)
<button
  onClick={() => setShowEditForm(true)}
  className="flex-1 flex items-center justify-center gap-2 px-4 py-3 text-white rounded-lg font-medium"
  style={{ backgroundColor: '#00b4d8' }}
>
  <Pencil className="w-4 h-4" />
  Редактировать
</button>
```

### 4.4 Проверка страницы /map (Решение Г)

**Текущая проблема:** Страница `/map` не передаёт `onPlaceClick`:

```tsx
// app/map/page.tsx:32-36
<YandexMap
  city={isAuthenticated ? user?.city : null}
  blurred={!isAuthenticated}
  onRegisterClick={handleRegisterClick}
  // ОТСУТСТВУЕТ: onPlaceClick={handlePlaceClick}
/>
```

**Это НЕ проблема для "Мои места", но нужно исправить для публичной карты.**

## 5. Acceptance Criteria

### AC1: Кнопки отображаются в модальном окне
- **Given** пользователь на странице `/profile` → вкладка "Мои места"
- **And** у пользователя есть сохранённые места
- **When** кликает на карточку места в списке
- **Then** открывается модальное окно
- **And** в заголовке видны иконки карандаша и корзины
- **And** внизу видны кнопки "Редактировать" и "Удалить"

### AC2: Редактирование работает
- **Given** модальное окно просмотра открыто
- **When** кликает на "Редактировать"
- **Then** открывается форма редактирования с предзаполнёнными данными
- **And** можно изменить любое поле
- **And** кнопка "Сохранить" отправляет PUT запрос

### AC3: Удаление работает
- **Given** модальное окно просмотра открыто
- **When** кликает на "Удалить"
- **Then** открывается диалог подтверждения
- **And** при подтверждении место удаляется
- **And** отображается уведомление "Место успешно удалено"
