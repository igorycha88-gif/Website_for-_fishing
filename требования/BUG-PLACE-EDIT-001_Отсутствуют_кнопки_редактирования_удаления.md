# Bug Report: Отсутствуют кнопки редактирования и удаления в модальном окне просмотра места

**ID**: BUG-PLACE-EDIT-001
**Priority**: High
**Severity**: Major
**Status**: New
**Date**: 2024-02-17
**Author**: Business Analyst

---

## Problem Statement

**Описание проблемы**: При клике на карточку места в разделе "Мои места" открывается модальное окно просмотра, но кнопки "Редактировать" и "Удалить" отсутствуют.

**Ожидаемое поведение**: В модальном окне просмотра места должны быть видны кнопки редактирования и удаления (согласно US-PLACE-EDIT-001).

**Фактическое поведение**: Модальное окно открывается, отображает информацию о месте, но кнопки действий отсутствуют.

### Возможные причины

#### 1. Проблема сборки/деплоя (ВЕРОЯТНО)
- Код на диске обновлен, но dev-сервер или production не перезапущен
- Кэш браузера содержит старую версию
- Не выполнен `npm run build` для production

#### 2. CSS/Стили (ВОЗМОЖНО)
- Tailwind классы не скомпилированы корректно
- Конфликт z-index с другими элементами
- Кнопки скрыты через стили

#### 3. Условие рендеринга (МАЛО ВЕРОЯТНО)
- Проблема с состоянием `selectedPlace`
- Проблема с булевыми флагами

## Solution

### Если проблема в сборке

```bash

npm run dev
```

### Если кнопки не рендерятся

```tsx
{selectedPlace && !showEditForm && !showDeleteConfirm && (
```

## Acceptance Criteria

**AC1: Кнопки видны**
- **Given** пользователь авторизован и имеет сохраненные места
- **When** кликает на карточку места
- **Then** открывается модальное окно с видимыми кнопками "Редактировать" и "Удалить"

**AC2: Редактирование работает**
- **Given** модальное окно просмотра открыто
- **When** кликает на кнопку "Редактировать"
- **Then** открывается форма редактирования с предзаполненными данными

**AC3: Удаление работает**
- **Given** модальное окно просмотра открыто
- **When** кликает на кнопку "Удалить"
- **Then** открывается диалог подтверждения удаления
