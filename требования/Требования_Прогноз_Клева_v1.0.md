## 1. Обзор

### 1.1. Описание функции

Система прогноза клева рыбы предоставляет пользователям персонализированные прогнозы активности рыбы на основе:
- Погодных условий (температура, давление, ветер, осадки)
- Времени года и суток
- Фазы луны
- Особенностей конкретных видов рыб
- Географического региона пользователя

### 1.2. Бизнес-ценность

**Для пользователей**:
- Планирование рыбалки в оптимальное время
- Выбор правильной стратегии ловли
- Экономия времени на неудачные выезды

**Для бизнеса**:
- Увеличение DAU/MAU (пользователи заходят за прогнозом)
- Дифференциация от конкурентов
- Удержание пользователей (retention)

### 3.2. Рекомендация: OpenWeatherMap

**Обоснование**:
1. **Проверенное решение** - самый популярный погодный API в мире
2. **Лимиты достаточны** - 85 регионов × 1 запрос = 85 req/день (в пределах 1000)
3. **Отличная документация** - легкая интеграция
4. **Надежность** - 99.9% uptime SLA
5. **Все нужные данные** - давление, ветер, осадки, фаза луны

**Fallback**: Open-Meteo как резервный (безлимитный)

## 4. Функциональные требования

### 4.1. Новые таблицы БД

#### 4.1.1. regions - Регионы России

```sql
CREATE TABLE regions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL UNIQUE,
    code VARCHAR(10) NOT NULL UNIQUE,
    latitude NUMERIC(10, 8) NOT NULL,
    longitude NUMERIC(11, 8) NOT NULL,
    timezone VARCHAR(50) NOT NULL DEFAULT 'Europe/Moscow',
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX idx_regions_code ON regions(code);
CREATE INDEX idx_regions_active ON regions(is_active);
```

#### 4.1.2. weather_data - Погодные данные

```sql
CREATE TABLE weather_data (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    region_id UUID NOT NULL REFERENCES regions(id) ON DELETE CASCADE,
    forecast_date DATE NOT NULL,
    forecast_hour INTEGER NOT NULL CHECK (forecast_hour >= 0 AND forecast_hour <= 23),
    temperature NUMERIC(5, 2),
    feels_like NUMERIC(5, 2),
    pressure_hpa INTEGER,
    humidity INTEGER,
    wind_speed NUMERIC(5, 2),
    wind_direction INTEGER,
    wind_gust NUMERIC(5, 2),
    cloudiness INTEGER,
    precipitation_mm NUMERIC(5, 2),
    precipitation_probability INTEGER,
    weather_condition VARCHAR(50),
    weather_icon VARCHAR(20),
    visibility_m INTEGER,
    uv_index NUMERIC(3, 1),
    moon_phase NUMERIC(3, 2),
    sunrise TIME,
    sunset TIME,
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(region_id, forecast_date, forecast_hour)
);

CREATE INDEX idx_weather_region_date ON weather_data(region_id, forecast_date);
CREATE INDEX idx_weather_date ON weather_data(forecast_date);
```

#### 4.1.3. fishing_forecasts - Прогнозы клева

```sql
CREATE TABLE fishing_forecasts (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    region_id UUID NOT NULL REFERENCES regions(id) ON DELETE CASCADE,
    fish_type_id UUID NOT NULL REFERENCES fish_types(id) ON DELETE CASCADE,
    forecast_date DATE NOT NULL,
    time_of_day VARCHAR(20) NOT NULL CHECK (time_of_day IN ('morning', 'day', 'evening', 'night')),
    bite_score INTEGER NOT NULL CHECK (bite_score >= 0 AND bite_score <= 100),
    temperature_score INTEGER,
    pressure_score INTEGER,
    wind_score INTEGER,
    moon_score INTEGER,
    precipitation_score INTEGER,
    recommendation TEXT,
    best_baits TEXT[],
    best_depth VARCHAR(50),
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(region_id, fish_type_id, forecast_date, time_of_day)
);

CREATE INDEX idx_forecast_region_date ON fishing_forecasts(region_id, forecast_date);
CREATE INDEX idx_forecast_fish ON fishing_forecasts(fish_type_id);
CREATE INDEX idx_forecast_date_score ON fishing_forecasts(forecast_date, bite_score DESC);
```

#### 4.1.4. fish_bite_settings - Настройки клева по видам рыб

> **Примечание**: Параметры задаются как seed-данные при инициализации БД. Редактирование через UI не требуется на данном этапе.

```sql
CREATE TABLE fish_bite_settings (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    fish_type_id UUID NOT NULL UNIQUE REFERENCES fish_types(id) ON DELETE CASCADE,
    optimal_temp_min NUMERIC(5, 2) NOT NULL DEFAULT 10,
    optimal_temp_max NUMERIC(5, 2) NOT NULL DEFAULT 25,
    optimal_pressure_min INTEGER NOT NULL DEFAULT 750,
    optimal_pressure_max INTEGER NOT NULL DEFAULT 770,
    max_wind_speed NUMERIC(5, 2) NOT NULL DEFAULT 8,
    precipitation_tolerance INTEGER NOT NULL DEFAULT 2,
    prefer_morning BOOLEAN DEFAULT true,
    prefer_evening BOOLEAN DEFAULT true,
    prefer_overcast BOOLEAN DEFAULT false,
    active_in_winter BOOLEAN DEFAULT false,
    moon_sensitivity NUMERIC(3, 2) DEFAULT 0.5,
    season_start_month INTEGER DEFAULT 4,
    season_end_month INTEGER DEFAULT 10,
    region_ids UUID[] DEFAULT '{}',  -- Список ID регионов где водится эта рыба
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Индекс для поиска рыб по региону
CREATE INDEX idx_fish_bite_settings_regions ON fish_bite_settings USING GIN(region_ids);
```

**Важно**: Поле `region_ids` содержит массив UUID регионов, в которых водится данный вид рыбы. Это позволяет фильтровать список рыб при отображении прогноза — показывать только те виды, которые обитают в выбранном регионе.

### 4.3. Seed данные настроек клева по видам рыб

> **Важно**: Поле `region_ids` определяет в каких регионах показывать рыбу в прогнозе. 
> Если рыба водится везде — массив пустой `{}`
> Если только в определенных регионах — перечислены ID регионов

**Принципы распределения рыб по регионам**:
- **Повсеместные** (пустой массив) — водятся почти везде в европейской части России

- **Южные** — Краснодар, Крым, Ростов, Астрахань

## 6. Логика прогнозирования

### 6.1. Алгоритм расчета bite_score

```
bite_score = (
    temperature_score * 0.25 +
    pressure_score * 0.25 +
    wind_score * 0.20 +
    moon_score * 0.15 +
    precipitation_score * 0.10 +
    time_of_day_bonus * 0.05
)
```

### 6.2. Расчет компонентов

#### Temperature Score (0-100)
```python
def calculate_temperature_score(actual_temp, fish_settings):
    optimal_min = fish_settings.optimal_temp_min
    optimal_max = fish_settings.optimal_temp_max
    optimal_range = optimal_max - optimal_min
    
    if optimal_min <= actual_temp <= optimal_max:
        return 100
    elif actual_temp < optimal_min:
        diff = optimal_min - actual_temp
        return max(0, 100 - (diff / optimal_range) * 50)
    else:
        diff = actual_temp - optimal_max
        return max(0, 100 - (diff / optimal_range) * 50)
```

#### Pressure Score (0-100)
```python
def calculate_pressure_score(actual_pressure, fish_settings):
    optimal_min = fish_settings.optimal_pressure_min
    optimal_max = fish_settings.optimal_pressure_max
    
    if optimal_min <= actual_pressure <= optimal_max:
        return 100
    
    if actual_pressure < optimal_min:
        diff = optimal_min - actual_pressure
        return max(0, 100 - diff * 3)
    else:
        diff = actual_pressure - optimal_max
        return max(0, 100 - diff * 3)
```

#### Wind Score (0-100)
```python
def calculate_wind_score(wind_speed, fish_settings):
    max_wind = fish_settings.max_wind_speed
    
    if wind_speed <= max_wind * 0.5:
        return 100
    elif wind_speed <= max_wind:
        return 100 - (wind_speed / max_wind) * 30
    else:
        return max(0, 70 - (wind_speed - max_wind) * 5)
```

#### Moon Score (0-100)
```python
def calculate_moon_score(moon_phase, fish_settings):
    sensitivity = fish_settings.moon_sensitivity
    
    # moon_phase: 0=new moon, 0.5=full moon, 1=new moon
    # Лучшая активность: новолуние (0, 1) и полнолуние (0.5)
    distance_to_extreme = min(abs(moon_phase), abs(moon_phase - 0.5), abs(moon_phase - 1))
    
    base_score = 100 - distance_to_extreme * 100
    return base_score * sensitivity + 50 * (1 - sensitivity)
```

#### Time of Day Bonus
```python
TIME_PERIODS = {
    'morning': (5, 10),   # 05:00 - 10:00
    'day': (10, 17),      # 10:00 - 17:00
    'evening': (17, 22),  # 17:00 - 22:00
    'night': (22, 5)      # 22:00 - 05:00
}

def calculate_time_bonus(hour, fish_settings):
    time_of_day = get_time_of_day(hour)
    
    if time_of_day == 'morning' and fish_settings.prefer_morning:
        return 100
    elif time_of_day == 'evening' and fish_settings.prefer_evening:
        return 100
    elif time_of_day == 'day':
        return 60
    else:
        return 30
```

### 6.3. Генерация рекомендаций

```python
def generate_recommendation(bite_score, weather, fish_settings):
    recommendations = []
    
    if bite_score >= 80:
        base = "Отличный клев!"
    elif bite_score >= 60:
        base = "Хороший клев."
    elif bite_score >= 40:
        base = "Умеренный клев."
    else:
        base = "Слабый клев."
    
    # Специфические рекомендации
    if weather['wind_speed'] > 5:
        recommendations.append("Используйте более тяжелые приманки")
    
    if weather['cloudiness'] > 70 and fish_settings.prefer_overcast:
        recommendations.append("Облачность благоприятна")
    
    if weather['precipitation'] > 0:
        recommendations.append("Возможен дождь, подготовьте снасти")
    
    return base + " " + " ".join(recommendations)
```

# Cron: 0 1 * * * (каждый день в 01:00 MSK)

async def fetch_weather_data():
    regions = await get_all_active_regions()
    
    for region in regions:
        # Получаем прогноз на 4 дня (48 часов детально + 3 дня daily)

            lat=region.latitude,

        )
        
        # Сохраняем почасовые данные
        for hour_data in weather_data.hourly:
            await save_weather_data(region.id, hour_data)
        
        # Сохраняем daily данные
        for day_data in weather_data.daily:

### 7.2. Расчет прогнозов после сбора данных

```python
async def calculate_all_forecasts():
    regions = await get_all_active_regions()
    fish_types = await get_all_active_fish_types()
    
    for region in regions:
        for fish_type in fish_types:
            settings = await get_fish_bite_settings(fish_type.id)
            
            # Фильтрация: пропускаем рыбу, если она не водится в этом регионе
            # Пустой массив region_ids = {} означает повсеместное распространение
            if settings.region_ids and region.id not in settings.region_ids:
                continue
            
            for day_offset in range(4):  # Сегодня + 3 дня
                forecast_date = today + timedelta(days=day_offset)
                weather = await get_weather_for_date(region.id, forecast_date)
                
                for time_of_day in ['morning', 'day', 'evening', 'night']:
                    score = calculate_bite_score(weather, settings, time_of_day)
                    await save_forecast(region.id, fish_type.id, forecast_date, time_of_day, score)
```

### 7.3. Получение прогноза с фильтрацией по региону

```python
async def get_forecast_for_region(region_id: UUID, date: date):
    # Получаем все прогнозы для региона и даты
    forecasts = await db.execute(
        select(FishingForecast, FishBiteSettings, FishType)
        .join(FishBiteSettings, FishingForecast.fish_type_id == FishBiteSettings.fish_type_id)
        .join(FishType, FishBiteSettings.fish_type_id == FishType.id)
        .where(FishingForecast.region_id == region_id)
        .where(FishingForecast.forecast_date == date)
        .where(
            or_(
                FishBiteSettings.region_ids == {},  # Повсеместные
                FishBiteSettings.region_ids.contains([region_id])  # Водится в регионе
            )
        )
        .order_by(FishingForecast.bite_score.desc())
    )
    return forecasts
```

5. **Детализация по выбранной рыбе** (раскрывающийся блок)
6. **Рекомендации**

**Фильтрация рыб по региону**:

## 9. Non-Functional Requirements

### 9.1. Performance
- **API Response**: < 200ms для GET запросов прогноза
- **Scheduled Task**: сбор данных для 85 регионов < 5 минут
- **Database**: индексы на всех частых запросах

### 9.2. Reliability
- **Fallback API**: Open-Meteo как резерв
- **Retry Logic**: 3 попытки при ошибке API
- **Data Retention**: погодные данные хранятся 30 дней

### 9.3. Caching
- **Redis**: кэширование прогнозов на 1 час
- **Cache Key**: `forecast:{region_id}:{date}:{fish_type_id}`
