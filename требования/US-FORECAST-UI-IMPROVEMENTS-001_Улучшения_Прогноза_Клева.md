## 1. Обзор

### 1.1. Описание

Данный документ описывает улучшения UI/UX в разделе "Прогноз клева" на вкладке "Мои места":
1. Отображение типа луны возле значка с корректными восходом/закатом по UTC
2. Исправление проблемы с маленьким окном при отсутствии данных по региону
3. Автоматическая пролонгация прогноза на 3 дня вперед
4. Цветовая индикация процента клева

### 1.2. Бизнес-ценность

**Для пользователей**:
- Более информативный прогноз с точными данными о луне
- Удобный выбор региона даже при отсутствии данных
- Актуальный прогноз на 3 дня вперед в любой момент
- Быстрая визуальная оценка качества клева по цвету

**Для бизнеса**:
- Улучшение UX и удовлетворенности пользователей
- Снижение обращений в поддержку
- Увеличение retention

## 2. User Stories

### US-1: Отображение типа луны и времени восхода/заката по UTC

**As a** пользователь,
**I want to** видеть тип луны (растущая, убывающая, полнолуние, новолуние) возле значка луны и время восхода/заката с учетом UTC,
**So that** я могу более точно планировать рыбалку с учетом фаз луны.

#### Priority
- [x] High (MVP)

#### Actors
- [x] Зарегистрированный пользователь
- [x] Незарегистрированный посетитель

#### Acceptance Criteria

**AC1: Отображение типа луны**
- **Given** я нахожусь на вкладке "Мои места" в разделе "Прогноз клева"
- **When** отображается блок текущей погоды
- **Then** возле значка луны в скобках отображается тип луны:
  - "Растущая" (фаза 0.01 - 0.49)
  - "Убывающая" (фаза 0.51 - 0.99)
  - "Полнолуние" (фаза 0.45 - 0.55)
  - "Новолуние" (фаза 0.0 - 0.05 или 0.95 - 1.0)

**AC2: Время восхода/заката по UTC**
- **Given** отображается время восхода и заката
- **When** данные получены от API
- **Then** время отображается в часовом поясе региона (timezone из таблицы regions)
- **And** время конвертируется из UTC в локальное время региона

#### Технические детали

**Frontend изменения** (`frontend/types/forecast.ts`):
```typescript
export function getMoonPhaseType(phase: number | null): string {
  if (phase === null) return '';
  
  if (phase <= 0.05 || phase >= 0.95) return 'Новолуние';
  if (phase >= 0.45 && phase <= 0.55) return 'Полнолуние';
  if (phase < 0.5) return 'Растущая';
  return 'Убывающая';
}
```

**Frontend изменения** (`frontend/components/FishingForecast.tsx`):
- Добавить отображение типа луны в скобках: `🌙 Луна (Растущая)`
- Конвертировать sunrise/sunset из UTC в локальное время региона

**Backend**: timezone уже хранится в таблице `regions`, sunrise/sunset сохраняются как time в UTC.

### US-2: Исправление маленького окна при отсутствии данных по региону

**As a** пользователь,
**I want to** видеть полный выпадающий список регионов с поиском даже когда по выбранному региону нет данных,
**So that** я могу легко найти и выбрать другой регион.

#### Priority
- [x] High (MVP)

#### Actors
- [x] Зарегистрированный пользователь
- [x] Незарегистрированный посетитель

#### Acceptance Criteria

**AC1: Минимальная высота dropdown**
- **Given** я открываю dropdown выбора региона
- **When** загружаются данные или произошла ошибка
- **Then** dropdown имеет минимальную высоту, достаточную для отображения:
  - Поля поиска
  - Минимум 5 регионов в списке
  - Или сообщения "Нет данных"

**AC2: Список регионов виден при поиске**
- **Given** dropdown открыт
- **When** я ввожу текст в поле поиска
- **Then** список найденных регионов отображается полностью
- **And** список не обрезается снизу

**AC3: Сообщение при отсутствии данных**
- **Given** выбран регион без данных прогноза
- **When** отображается компонент прогноза
- **Then** показывается сообщение "Данные прогноза для этого региона временно недоступны"
- **And** пользователь может выбрать другой регион

#### Технические детали

**Frontend изменения** (`frontend/components/FishingForecast.tsx`):

```tsx

// Добавить состояние "нет данных" вместо пустого блока

    </p>
  </div>

```

### US-3: Автоматическая пролонгация прогноза на 3 дня

**As a** система,
**I want to** автоматически обновлять прогноз на 3 дня вперед при смене даты,
**So that** пользователи всегда имеют актуальный прогноз.

#### Priority
- [x] High (MVP)

#### Actors
- [x] System (автоматизированные процессы)

#### Acceptance Criteria

**AC1: Scheduled task в 01:00 MSK**
- **Given** наступает 01:00 MSK
- **When** срабатывает scheduled task
- **Then** система собирает данные на текущий день + 3 дня вперед
- **And** удаляет устаревшие данные (дата < сегодня)

**AC2: Сдвиг прогноза при смене даты**
- **Given** наступает новый день (00:00 по каждому timezone региона)
- **When** пользователь открывает прогноз
- **Then** прогноз отображается на сегодня + 3 дня вперед
- **And** вчерашний прогноз не отображается

**AC3: Загрузка тестовых данных**
- **Given** требуется тестирование
- **When** выполняется загрузка данных
- **Then** загружаются данные для:
  - Московская область (code: MOS)
  - Ленинградская область (code: LEN)
  - На 3 дня вперед от текущей даты

#### Технические детали

**Backend**: Текущая реализация scheduler уже собирает данные на 4 дня (сегодня + 3):
- 01:00 MSK - основной запуск
- 13:00 MSK - дополнительный запуск для обновления

**API Endpoint для ручной загрузки**:
```
POST /api/v1/weather/collect
{
  "region_codes": ["MOS", "LEN"],
  "days": 3
}
```

**Seed данные регионов** - убедиться что в таблице `regions` есть:
```sql
INSERT INTO regions (name, code, latitude, longitude, timezone) VALUES
('Москва', 'MOW', 55.7558, 37.6173, 'Europe/Moscow'),
('Московская область', 'MOS', 55.7558, 37.6173, 'Europe/Moscow'),
('Ленинградская область', 'LEN', 59.9391, 30.3158, 'Europe/Moscow');
```

### US-4: Цветовая индикация процента клева

**As a** пользователь,
**I want to** видеть цветовую индикацию рядом с процентом клева,
**So that** я могу быстро оценить качество клева без чтения текста.

#### Priority
- [x] High (MVP)

#### Actors
- [x] Зарегистрированный пользователь
- [x] Незарегистрированный посетитель

#### Acceptance Criteria

**AC1: Цветовая полоса для каждого процента**
- **Given** отображается процент клева для рыбы
- **When** bite_score находится в диапазоне
- **Then** отображается цветная полоса (progress bar) соответствующего цвета:

| Диапазон | Цвет | Label |
|----------|------|-------|
| 80-100% | Зеленый (#22c55e) | Отлично |
| 65-79% | Салатовый (#84cc16) | Хорошо |
| 50-64% | Желтый (#eab308) | Умеренно |
| 35-49% | Оранжевый (#f97316) | Слабо |
| 0-34% | Красный (#ef4444) | Плохо |

**AC2: Цвет текста процента**
- **Given** отображается процент клева
- **When** bite_score находится в диапазоне
- **Then** цвет текста совпадает с цветом полосы

**AC3: Применение ко всем местам отображения**
- **Given** прогноз отображается в любом месте UI
- **When** показывается bite_score
- **Then** цветовая индикация применяется в:
  - Список рыб (ТОП клев сегодня)
  - Детальный прогноз по времени суток
  - Прогноз на 3 дня (multi_day_forecast)

#### Технические детали

**Frontend**: Функции уже реализованы в `frontend/types/forecast.ts`:
```typescript
export function getBiteScoreColor(score: number): string {
  if (score >= 80) return 'bg-green-500';
  if (score >= 65) return 'bg-lime-500';
  if (score >= 50) return 'bg-yellow-500';
  if (score >= 35) return 'bg-orange-500';
  return 'bg-red-500';
}

export function getBiteScoreTextColor(score: number): string {
  if (score >= 80) return 'text-green-600';
  if (score >= 65) return 'text-lime-600';
  if (score >= 50) return 'text-yellow-600';
  if (score >= 35) return 'text-orange-600';
  return 'text-red-600';
}
```

**Проблема**: Цветовая полоса не отображается или отображается только красная. Требуется проверка:
1. Убедиться что progress bar имеет правильную ширину
2. Проверить CSS классы Tailwind

## 3. Non-Functional Requirements

### 3.1. Performance
- **UI Response**: Dropdown должен открываться < 100ms
- **API Response**: Прогноз загружается < 200ms
- **Scheduled Task**: Сбор данных для всех регионов < 5 минут

### 3.2. Reliability
- **Fallback**: При отсутствии данных показывать сообщение, а не пустой блок
- **Retry**: Автоматический retry при ошибке API (уже реализовано через tenacity)

### 3.3. UX
- **Минимальная высота dropdown**: 280px для отображения поиска + минимум 5 элементов
- **Цветовая индикация**: Понятная визуальная градация от зеленого до красного

## 4. API Changes

### 4.1. Добавить поле moon_phase_type в ответ API

**Endpoint**: `GET /api/v1/forecast/:region_id`

**Текущий ответ**:
```json
{
  "weather": {
    "moon_phase": 0.25,
    "sunrise": "07:30:00",
    "sunset": "18:45:00"
  }
}
```

**Новый ответ**:
```json
{
  "weather": {
    "moon_phase": 0.25,
    "moon_phase_type": "Растущая",
    "sunrise": "07:30:00",
    "sunset": "18:45:00",
    "timezone": "Europe/Moscow"
  }
}
```

### 4.2. Endpoint для ручной загрузки данных (если не существует)

**Endpoint**: `POST /api/v1/weather/collect`

**Request**:
```json
{
  "region_codes": ["MOS", "LEN"],
  "days": 3
}
```

**Response**:
```json
{
  "status": "success",
  "collected": 2,
  "total_records": 48
}
```
