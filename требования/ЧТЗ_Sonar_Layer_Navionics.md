## 1. Цели и задачи

### 1.1 Бизнес-цель
Создать профессиональный слой глубин на карте (в стиле Navionics Marine Charts), чтобы рыбак мог включить отдельный слой, увидеть цветовую схему глубин и прочитать числовые обозначения глубин прямо на карте.

### 1.2 Пользовательская ценность
- Рыбак включает слой «Глубины» и видит **цветные полигоны водоёмов** — каждый цвет = диапазон глубины
- На карте отображаются **метки глубин** (числа в метрах) для каждого крупного водоёма
- Береговая линия водоёмов обведена **контуром** (изобата 0м)
- Для морских/океанских зон — **кастомные тайлы** GEBCO, перекрашенные под нашу легенду
- Панель слоёв предлагает **несколько цветовых схем** (Navionics, Стандартная, Контрастная)

### 1.3 Метрики успеха
- Endpoint `/api/v1/depth/areas` отвечает < 100ms (при закэшированных полигонах)
- Endpoint `/api/v1/depth/labels` отвечает < 50ms
- Полигоны и метки загружаются < 500ms после движения карты
- Кастомные тайлы перекрашиваются < 200ms на тайл (с кэшированием)

## 2. Функциональные требования

### 2.1 User Stories

**US-SLR-001: Цветные полигоны глубин**
**Как** рыбак, **я хочу** видеть цветные полигоны водоёмов на карте, **чтобы** визуально оценить глубину по цвету.
- Given: пользователь на странице `/map`
- When: включает тумблер «Глубины» в панели слоёв
- Then: поверх базовой карты появляются **полупрозрачные цветные полигоны** водоёмов
- And: каждый полигон окрашен по нашей цветовой схеме (0-2м, 2-5м, 5-10м, 10-20м, 20-50м, >50м)
- And: форма полигона соответствует реальному контуру водоёма (из OSM)
- And: есть слайдер прозрачности (0-100%, по умолчанию 60%)

**US-SLR-002: Метки глубин**
**Как** рыбак, **я хочу** видеть числовые метки глубин на карте, **чтобы** прочитать точную глубину водоёма.
- Given: слой глубин включён
- When: пользователь смотрит на карту
- Then: в центре каждого крупного водоёма отображается **метка с глубиной** (например «24м», «5.2м»)
- And: метки не накладываются друг на друга при зуме
- And: метки скрываются при зуме < 7 (мировой масштаб)
- And: есть тумблер «Метки» для включения/выключения

**US-SLR-003: Контур водоёмов (изобаты)**
**Как** рыбак, **я хочу** видеть контурную линию вокруг водоёмов, **чтобы** чётко видеть границы воды и суши.
- Given: слой глубин включён
- When: тумблер «Изобаты» включён
- Then: каждый полигон водоёма обведён **контрастной линией** по береговой линии
- And: цвет линии зависит от цветовой схемы

**US-SLR-004: Цветовые схемы**
**Как** рыбак, **я хочу** выбрать цветовую схему глубин, **чтобы** настроить отображение под свои предпочтения.
- Given: слой глубин включён
- When: пользователь нажимает на селектор цветовой схемы
- Then: появляется список схем: «Navionics» (синий), «Контрастная» (радужная), «Спорт» (зелёный)
- And: при выборе схемы все полигоны и тайлы перекрашиваются мгновенно

**US-SLR-005: Кастомные тайлы для морских зон**
**Как** рыбак, **я хочу** видеть глубины морей и океанов в нашей цветовой схеме, **чтобы** цветовая схема была единой.
- Given: слой глубин включён
- When: пользователь перемещается к морю/океану
- Then: появляются **перекрашенные тайлы** GEBCO в нашей цветовой схеме
- And: цвета тайлов **совпадают с легендой** в панели слоёв
- And: тайлы кэшируются для повторного использования

**US-SLR-006: Интерактивная легенда**
**Как** рыбак, **я хочу** видеть легенду глубин с понятными обозначениями, **чтобы** правильно интерпретировать цвета.
- Given: слой глубин включён
- When: пользователь смотрит на панель слоёв
- Then: отображается цветная легенда с диапазонами: 0-2м, 2-5м, 5-10м, 10-20м, 20-50м, >50м
- And: легенда обновляется при смене цветовой схемы

### 2.2 Нефункциональные требования
- **Производительность:** viewport-based загрузка (только видимые полигоны/метки)
- **Кэширование:** Redis для результатов запросов полигонов, файловый кэш для тайлов
- **Адаптивность:** панель слоёв работает на mobile (375px) и desktop (1280px)
- **Логирование:** structlog на каждом endpoint (info на вход, info на выход, error при ошибке)

## 3. Технические требования

### 3.1 База данных

**Новая таблица `water_body_polygons`** (миграция 011):
```sql
CREATE TABLE water_body_polygons (
    id SERIAL PRIMARY KEY,
    name VARCHAR(500) NOT NULL,
    water_type VARCHAR(20) DEFAULT 'lake',
    geometry JSONB NOT NULL,          -- GeoJSON Polygon/MultiPolygon coordinates [[lon, lat], ...]
    max_depth FLOAT,
    avg_depth FLOAT,
    area_km2 FLOAT,
    source VARCHAR(20) DEFAULT 'OSM',
    region VARCHAR(200),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_wbp_geometry_gist ON water_body_polygons USING GIST (geometry);
CREATE INDEX idx_wbp_name ON water_body_polygons (name);
```

### 3.2 Backend (depth-service)

**Новые модули:**
- `app/services/osm_polygon_importer.py` — импорт полигонов из OSM Overpass
- `app/services/depth_areas.py` — viewport-based запрос полигонов
- `app/services/depth_labels.py` — генерация меток глубин
- `app/services/tile_recolor.py` — перекраска GEBCO тайлов

**Новые endpoints:**
- `GET /api/v1/depth/areas?minLat=&minLon=&maxLat=&maxLon=` → GeoJSON FeatureCollection
- `GET /api/v1/depth/labels?minLat=&minLon=&maxLat=&maxLon=&zoom=` → GeoJSON Point features
- `GET /api/v1/depth/contours?minLat=&minLon=&maxLat=&maxLon=` → GeoJSON LineString features

**Изменяемые endpoints:**
- `GET /api/v1/depth/tiles/{z}/{x}/{y}.png` — добавить перекраску под нашу цветовую схему

**Цветовая схема (единая константа):**
```python
DEPTH_COLOR_RAMP = [
    (0, 2,    "#B3E5FC"),  # Очень мелко — светло-голубой
    (2, 5,    "#4FC3F7"),  # Мелководье — голубой
    (5, 10,   "#0288D1"),  # Средняя глубина — синий
    (10, 20,  "#01579B"),  # Глубоко — тёмно-синий
    (20, 50,  "#1A237E"),  # Очень глубоко — индиго
    (50, 9999,"#000C2E"),  # Бездна — почти чёрный
]
```

### 3.3 Frontend

**Изменяемые файлы:**
- `frontend/components/YandexMap.tsx` — рендер полигонов, меток, контуров
- `frontend/components/LayersPanel.tsx` — новые контролы (схемы, метки, изобаты)
- `frontend/types/depth.ts` — новые типы (DepthArea, DepthLabel, ColorScheme)
- `frontend/hooks/useDepth.ts` — расширить хук

**Новые хуки:**
- `frontend/hooks/useDepthAreas.ts` — загрузка полигонов по viewport
- `frontend/hooks/useDepthLabels.ts` — загрузка меток по viewport

**Рендеринг:**
- `ymaps.Polygon` для каждого водоёма (fill = цвет по глубине, stroke = изобата)
- `ymaps.Placemark` с текстом глубины для меток
- Debounced обновление при `boundschange` (300ms)

## 6. Риски

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Overpass API медленный/недоступен при seed | Средняя | Retry + timeout + fallback на bbox |
| Полигоны OSM неполные для некоторых водоёмов | Средняя | Fallback на bbox из ru_water_bodies |
| Перекраска тайлов медленная | Низкая | Файловый кэш + async |
| Большое количество полигонов в viewport | Низкая | LIMIT в SQL + упрощение геометрии |
